class CornerPositionEnum(value)[source]

Bases: Enum

Represents a corner.

Corner positions are stored as coordinates in the coordinate system with: - the origin on the upper-left corner - the X-axis that goes from the origin to the upper-right corner - the Y-axis that goes from the origin to the lower-left corner

LOWER_LEFT = Position(x=0, y=1)
LOWER_RIGHT = Position(x=1, y=1)
UPPER_LEFT = Position(x=0, y=0)
UPPER_RIGHT = Position(x=1, y=0)

class TemplateRelativePositionEnum(value)[source]

Bases: Enum

Represent a relative position between two Template instances.

Template relative positions are stored as the relative displacement needed to encode the position. For example, LEFT_OF is encoded as (-1, 0) as a template on the left is one unit backwards on the X-axis.

ABOVE_OF = Position(x=0, y=-1)
BELOW_OF = Position(x=0, y=1)
LEFT_OF = Position(x=-1, y=0)
RIGHT_OF = Position(x=1, y=0)

to_numpy_shape() → tuple[int, int][source]

Returns the shape according to numpy indexing.

In the coordinate system used in this library, numpy indexes arrays using (y, x) coordinates. This method is here to translate a Shape instance to a numpy shape transparently for the user.
